Epics & Stories

Problem statements grouped into epics, broken into stories with effort and timing.

How to read this page

Each epic is a category of problem we're solving. Stories under each epic are concrete units of work. Every story carries a title, description, effort estimate, start date, and projected end date.

The Library catalog organizes finished agents by function (Marketing, QA, Development) for GTM. The epics below organize the work by problem domain — which is why QE and ADLC are tracked as separate epics even though both feed the Development and QA categories of the Library.

Epics

Epic 01 — QE agent workflow Shipping

A multi-agent QE workflow on Zebrunner, designed to fully replicate the default QE flow end to end across seven agents. Five are shipped: test case generation, normalization, code generation, execution, and self-healing. Failure analysis with notification, and report generation, are the remaining two. Mobile build was completed most recently; Jira integration is live — moving a ticket to the right state triggers the workflow.

Built on LangGraph rather than locked to a single LLM SDK, so the underlying model can be swapped without re-architecting. Packaged as a standalone solution so it can plug into any Zebrunner-compatible stack.

AgentStatusWhat it does
Test case generation Shipped Takes a user story and produces test case documentation in the format manual QE would have written by hand.
Normalization Shipped Evaluates whether a generated test case can be automated. Uses Playwright MCP and browser-use to walk the manual steps and pull metadata from the browser.
Code generation Shipped Generates the automation code for cases the normalization agent flagged as automatable. Opens the result as a pull request.
Execution Shipped Runs the generated test against the scalable Zebrunner grid. Reports results back into Zebrunner for tracking.
Self-healing Shipped Detects and repairs broken tests automatically when the underlying page or flow changes.
Failure analysis & notification In build Packages screenshots, full stack trace, video, and self-healing attempts into a structured report so a developer can fix the issue without context-switching.
Report generation In build Produces release and regression reports on demand via the Zebrunner MCP — replacing the dashboards layer with LLM-driven, customer-shaped output.

Built on LangGraph rather than locked to a single LLM SDK, so the underlying model can be swapped without re-architecting. Jira integration is live — moving a ticket to the right state triggers the workflow. Next horizon: extending coverage from functional testing into non-functional — performance, scalability, chaos engineering.
